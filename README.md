# Wareon Analytics

**Система аналитики нового поколения** — Telegram-бот, который не просто считает цифры, а помогает принимать бизнес-решения.

## Направления

| Модуль | Что умеет |
|---|---|
| 📊 **Бизнес-аналитика** | учёт продаж, прибыль и маржа, конверсия, средний чек, ROI/ROMI, точка безубыточности, расчёт зарплаты, воронка продаж |
| 📣 **Аналитика соцсетей** | добавьте бота в канал или группу — он собирает статистику: сообщения, посты, подписки/отписки, активные участники, прирост |
| 🛍 **Маркетплейсы** | воронка карточки (CTR → корзина → заказ), ДРР, юнит-экономика — для дизайнеров карточек и менеджеров МП |
| 📑 **Умные таблицы** | пришлите .xlsx/.csv — бот разберёт структуру, посчитает суммы/средние, найдёт топ-категории; после загрузки отвечает на вопросы обычным текстом: «топ товаров по выручке», «какой товар принёс больше всего?» |
| 📋 **Отчёты** | сводка за период со сравнением с прошлым периодом (▲/▼), лучшим днём недели, графиком и рекомендацией; регулярные отчёты по расписанию и алерты по марже |

## Быстрый старт

```bash
# 1. Установить зависимости (Python 3.11+)
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

# 2. Настроить токен
cp .env.example .env
# впишите BOT_TOKEN из @BotFather

# 3. Запустить
python -m wareon.main
```

## Основные команды бота

```
/start                          — меню и справка по разделам
/sale 15000 8000 сайт           — записать продажу
/profit 100000 60000            — прибыль, маржа, наценка
/conversion 1000 37             — конверсия
/avg 150000 42                  — средний чек
/roi 50000 200000               — ROI
/romi 300000 100000             — ROMI
/breakeven 100000 1500 900      — точка безубыточности
/salary 50000 800000 5 10000    — зарплата: оклад + % с продаж + бонус
/funnel показы:10000 клики:800 заказы:56 — воронка продаж
/card 50000 2500 400 120        — воронка карточки маркетплейса
/unit 1500 20 80 600 50         — юнит-экономика МП
/report 30                      — отчёт с графиком за 30 дней
/subscribe daily 09:00          — ежедневный отчёт по расписанию (МСК)
/subscribe weekly 10:00         — еженедельный отчёт по понедельникам
/unsubscribe                    — отключить регулярные отчёты
/alert 20                       — алерт, если маржа за сутки ниже 20%
/channels                       — статистика подключённых каналов/групп
/stats                          — статистика (внутри группы)
/tables                         — история загруженных таблиц
```

После загрузки таблицы вопросы задаются обычным текстом в личке:
«сумма выручки», «средняя цена», «топ товаров по выручке», «сколько строк».

Для аналитики соцсетей добавьте бота администратором в канал или группу — сбор статистики начнётся автоматически.

## Мини-приложение (Web App)

Визуальный дашборд в стилистике чёрный / белый / золото: экраны **Пульс бизнеса**,
**Соцсети**, **Маркетплейсы**, **ИИ-мозг** с нижней навигацией и тумблером Просто/Pro.
Файлы — в `webapp/`, данные пока демонстрационные.

Как подключить к боту:

1. Включить GitHub Pages: **Settings → Pages → Build and deployment → Source: GitHub Actions**.
   После пуша в `main` workflow `deploy-webapp.yml` опубликует дашборд на
   `https://<owner>.github.io/<repo>/`.
2. Прописать этот адрес в переменную окружения `WEBAPP_URL` (в панели хостинга бота).
3. После перезапуска в боте появится кнопка **«✨ Открыть дашборд»**, а кнопка-меню
   Telegram у поля ввода будет запускать мини-приложение.

Любой другой HTTPS-хостинг (Netlify, Vercel, свой сервер) тоже подойдёт — важно лишь,
чтобы `WEBAPP_URL` указывал на `index.html` по HTTPS.

## Реальные данные (API / бэкенд)

Чтобы дашборд показывал не демо, а настоящие цифры клиента, работает «склад данных» —
API на FastAPI (`wareon/api/`). Поток данных:

```
клиент шлёт таблицу боту → ingest.py распознаёт колонки → продажи в БД
дашборд открывается в Telegram → зовёт /api/pulse (с подписью Telegram)
API считает по данным этого клиента → дашборд показывает реальные цифры
```

Каждый запрос к API подписан Telegram Web App (`auth.py`), поэтому клиент видит только
свои данные — это заложено под продажу продукта нескольким клиентам.

Запуск API локально:

```bash
python -m wareon.api          # поднимет http://127.0.0.1:8000
# проверка: curl http://127.0.0.1:8000/api/health
```

Чтобы связать дашборд с API: разверните API на хостинге с HTTPS (Render, Railway,
свой сервер) с тем же `BOT_TOKEN`, затем впишите его адрес в `webapp/index.html`
в строку `window.WAREON_API = "..."`. Пусто — дашборд остаётся на демо-данных.

Для нескольких клиентов задайте `DATABASE_URL` на PostgreSQL (см. `.env.example`).

## Архитектура

```
wareon/
├── main.py           # точка входа: запуск polling
├── config.py         # настройки из .env
├── keyboards.py      # inline-меню
├── db/               # SQLite + SQLAlchemy (async): пользователи, продажи,
│                     # отслеживаемые чаты, события, загрузки таблиц
├── services/         # чистая бизнес-логика (покрыта тестами)
│   ├── analytics.py  # все бизнес-расчёты
│   ├── social.py     # агрегация статистики чатов
│   ├── tables.py     # разбор и сводка CSV/XLSX
│   ├── qa.py         # вопросы к таблице на естественном языке
│   ├── scheduler.py  # регулярные отчёты по расписанию (МСК)
│   ├── ingest.py     # приёмка таблиц: распознаёт колонки → продажи
│   └── reports.py    # отчёты, сравнение периодов, графики
├── api/              # «Склад данных» — FastAPI, отдаёт дашборду реальные цифры
│   ├── app.py        # эндпоинты (/api/pulse), CORS
│   ├── auth.py       # проверка подписи Telegram Web App (кто спрашивает)
│   └── schemas.py    # форматы ответов
├── webapp/           # Telegram Web App (мини-приложение), dark-tech визуал
│   └── index.html    # экраны Пульс / Соцсети / Маркет / ИИ-мозг + таб-бар
└── handlers/         # обработчики Telegram по разделам
```
