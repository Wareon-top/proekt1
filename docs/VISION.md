# Wareon Analytics — визия продукта и дорожная карта

> Живой документ. Фиксирует целевую картину SaaS-экосистемы и раскладывает её
> на инженерные этапы. Обновляется по мере проработки.

## Что это

SaaS-аналитика для бизнеса в Telegram (формат **Telegram Web Apps** + бот-ядро).
Визуал: премиальный dark-tech — чёрные фоны, бирюзовые акценты, бруталистичная
типографика. Слоган смысла: не просто считать цифры, а **подсказывать решения**.

## Четыре продуктовых блока + сервисная обвязка

### 📊 1. Умная бизнес-аналитика
- **Пульс бизнеса** — «заработано чистыми» на главном экране, динамика ▲/▼ к прошлой неделе.
- **Настраиваемые виджеты** — топ-5 прибыльных и антирейтинг убыточных.
- **Доходы / расходы / ФОТ** — все транзакции, зарплаты, аренда, накладные; диаграммы.
- **Бухгалтерия и банки** — загрузка выписок и автосортировка.
- **Налоговая копилка** — расчёт УСН/патента, виртуальное откладывание с поступлений, напоминания.
- **Дивидендная ёмкость** — сколько можно безопасно вывести без кассового разрыва.

### 📱 2. Соцсети
- **Базовые метрики** — подписчики, отписки, лайки, комментарии, вовлечённость по постам.
- **Умная атрибуция (ROMI)** — какой канал приносит живые продажи, а какой сливает бюджет.
- **Аналитика контента** — лучшее время постинга, какие форматы приносят деньги.

### 🎨 3. Маркетплейсы (спец-режим для дизайнеров)
- **ИИ-аудит визуала** — шум, читаемость, слепые зоны, цветовая температура обложки.
- **Симуляция выдачи и A/B** — карточка среди конкурентов, прогноз CTR до рекламы.
- **Премиальные HTML-отчёты** — чистые документы для селлера как аргумент экспертности.

### 🧠 4. Умный ИИ-мозг
- **Рецепты вместо алертов** — у каждой цифры: что это, норма по рынку, светофор, «как улучшить».
- **Шпионаж за конкурентами** — чужие остатки, суточная выручка, ценовые войны, пуши.
- **Кэширование советов** — тяжёлые ИИ-советы раз в сутки, сохраняются в системе.
- **Утренние сводки** — брифинг: итоги вчера + 3 приоритета на сегодня.

### ⚙️ Сервисная обвязка
- **Тумблер «Просто / Pro»** — новичкам цифры и стрелки, профи — когорты и юнит-экономика.
- **Мульти-аккаунты и роли** — владелец даёт менеджеру доступ только к заказам, скрыв прибыль.
- **Экспорт и API** — Excel, Google Sheets, корпоративные системы.
- **Тарифы** — вход от 1 990 ₽, +500 ₽ за дизайнерский модуль, корп-пакеты, Lifetime.

## Инженерная раскладка по тирам

Классификация по стоимости реализации, а не по важности.

### Тир A — строим на текущем стеке (aiogram + SQLite/Postgres), без внешних ключей
- Учёт доходов/расходов/ФОТ поверх существующей модели `Sale` (+ категории, расходы).
- Пульс бизнеса: «чистыми за период» + динамика — уже частично есть в `/report`.
- Топ прибыльных / антирейтинг — агрегация по товарам.
- Налоговая копилка: расчёт УСН 6%/15% и патента — чистая формула + напоминания (планировщик уже есть).
- Дивидендная ёмкость: формула от остатка кэша и обязательств.
- Базовые метрики соцсетей — расширение существующего трекинга чатов.
- Утренние сводки — уже есть каркас (`scheduler.py`), нужен текстовый брифинг.
- Экспорт в Excel — библиотека `openpyxl` уже в зависимостях.

### Тир B — нужен внешний доступ / ключи / оплата за API
- **ИИ-мозг** (рецепты, аудит визуала, брифинги, шпионаж) — нужен LLM-провайдер и бюджет на токены. Рекомендация: Claude API. Требует кэширования (раз в сутки), чтобы не жечь деньги.
- **API маркетплейсов** (WB, Ozon) — нужен API-ключ продавца; шпионаж за конкурентами — парсинг публичной выдачи (юридически серая зона, обсудить).
- **Банковские выписки** — форматы 1С/CSV банков; автоимпорт через API банка требует интеграции (Тинькофф/Точка дают API).
- **Соцсети кроме Telegram** (VK, блогеры) — API VK, ручной ввод для остального.

### Тир C — нужны продуктовые/бизнес-решения до кода
- **Web App vs бот-ядро** — реальный «тикающий счётчик» и настраиваемые виджеты возможны только в Telegram Web App (мини-приложение на HTML/JS), а не в сообщениях бота. Это развилка архитектуры (см. ниже).
- **Тарифы и оплата** — нужен платёжный провайдер (ЮKassa/Telegram Payments/CryptoBot) и модель подписок.
- **Мульти-аккаунты и роли** — модель прав доступа, влияет на всю схему БД. Заложить рано, реализовать по мере надобности.
- **Дизайн-система** — dark-tech визуал: нужен макет (Figma) до вёрстки Web App.

## Ключевая архитектурная развилка: бот ↔ Web App

Текущий MVP — чистый бот на командах (`/report`, `/sale`, …). Визия же описывает
**мини-приложение** (Web App): дашборды, тикающие счётчики, виджеты, тумблер Просто/Pro.

Это два разных фронтенда над **одним бэкендом**. Правильный путь:
1. Всю логику держать в сервисном слое (`services/`) — как сейчас.
2. Поверх сервисов сделать **HTTP API** (FastAPI).
3. Бот и Web App — два клиента этого API. Бот остаётся для команд и пушей, Web App — для дашбордов.

Так текущий код не выбрасывается, а становится фундаментом.

## Предлагаемая последовательность

1. **Расширить финансовый учёт** (расходы, категории, ФОТ) — фундамент «Пульса» и налогов. Тир A.
2. **Налоговая копилка + дивидендная ёмкость** — быстрые формулы, высокий «вау». Тир A.
3. **Вынести бэкенд в FastAPI** — подготовка к Web App без переписывания логики. Тир A/C.
4. **Подключить LLM (Claude)** для брифингов и «рецептов» с кэшированием. Тир B.
5. **Web App v1** — Пульс + виджеты на dark-tech дизайне. Тир C.
6. **Маркетплейсы и оплата** — по мере готовности ключей и бизнес-модели. Тир B/C.
